How the Algorithm Works

Step 1 — EDF Heuristic (Initial Schedule) Filter tasks with all dependencies met → sort by composite score Score = w₁ × (1/deadline) + w₂ × priority → assign to earliest available slot per statistician, respecting buffer and capacity constraints.
Step 2 — ILP Re-optimization Minimize Z = ∑Lᵢ subject to: assignment (each task once), capacity (one task per statistician per slot), completion time, lateness bounds, precedence, release dates, and daily meeting cap.
Step 3 — Stochastic Buffer Insertion Inflate scheduled durations: p'ᵢ = E[pᵢ] + α·σ(pᵢ). Default α = 1.5 means 90th-percentile coverage. Buffers absorb minor overruns without cascading lateness.
Step 4 — Dynamic Rescheduling If a task overruns buffer by >10 min, the engine re-optimizes only the affected window (same statistician, current + next day). Notifications are dispatched to all impacted scientists.
